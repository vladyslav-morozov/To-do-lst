# Інструкція для Claude Code: дизайн та UI AI-планера

## 0. Завдання

Підняти UI-скелет мобільного AI-планера дня у стилі Things 3 (Cultured Code). Mobile-first, портретна орієнтація. Десктоп — не пріоритет. Фінальна ціль — деплой на Vercel за публічним посиланням.

Серце продукту — два AI-дотики:
1. **Голосовий brain dump** → транскрипція в текст.
2. **AI-парсинг** неструктурованого тексту → структуровані задачі (назва, пріоритет, оцінка часу, дедлайн).

Усе інше — оболонка навколо цих двох дій.

---

## 1. Технологічний стек

- **Next.js 14+ (App Router) + TypeScript**.
- **Tailwind CSS** для стилів.
- **shadcn/ui** для базових примітивів (Button, Sheet, Dialog, Input).
- **framer-motion** для жестів і анімацій (drawer, swipe, expand).
- **lucide-react** для іконок.
- **zustand** (або useReducer + Context) для глобального стану задач — поки без бекенду, тримати в памʼяті + `localStorage`.
- Деплой — **Vercel**. Конфіг `next.config.mjs` мінімальний, без серверних баз даних на цьому етапі. AI-виклики — через Next.js API routes (`/api/transcribe`, `/api/parse`).

Стандарт коду: суворий TS, ESLint, Prettier. Усі компоненти — функціональні, без зайвих обгорток.

## 3. Layout shell

Один глобальний layout `app/layout.tsx`:

- Контейнер: `max-w-md mx-auto min-h-screen bg-white relative`.
- Усі екрани — клієнтські сторінки під цим контейнером.
- Зверху — **TopBar** (висота 56px, sticky): зліва `Menu` (≡) — відкриває drawer; справа — `Search` іконка (поки заглушка); по центру — назва екрану.
- Знизу праворуч — **FAB** (плаваючий мікрофон, `fixed bottom-6 right-5`, 64×64, синій, з тінню).
- **Drawer (бокове меню)** і **VoiceModal** (повноекранна запис-модалка) — рендеряться на рівні layout через portal/`Sheet`.

---

## 4. Бокове меню (Drawer)

- Викликається тапом по `Menu` у TopBar.
- **Ширина — 50% екрану**, висота — на весь viewport, фон `bg-white`, виїжджає зліва направо (`framer-motion`, ~250ms, easing `easeOut`).
- Права половина — `overlay` напівпрозорий, клік по ній закриває drawer.
- Закриття також — свайп вліво по самій панелі та повторний тап по `Menu` (кнопка лишається видимою всередині панелі зверху ліворуч).
- Усередині — вертикальний список:
  - Inbox / Today / Upcoming / Anytime / Someday / Logbook
  - Кожен рядок: іконка (за маппінгом 2.4) + назва (`text-base font-medium`) + лічильник справа (`text-secondary text-sm`). У Today — додатковий маленький червоний кружок з кількістю прострочених.
  - Активний пункт — фон `bg.muted`, текст `text.primary`.
- Тап по пункту → drawer закривається, екран змінюється.

Реалізація — `shadcn/ui` `Sheet` з `side="left"` і `className="w-1/2 sm:max-w-none"`.

## 5. Екрани (MVP)

### 5.1. Today (`/`)

Головний екран. Що бачить користувач при запуску.

- TopBar: `Menu` ліворуч, заголовок «Today», дата сірим під заголовком (`6 червня, неділя`).
- Якщо задач немає — empty state: велика іконка-зірка, текст «Натисни мікрофон і розкажи, що в тебе на сьогодні».
- Список задач, відсортований: спочатку **прострочені** (червоним заголовок секції «Overdue»), потім «Today».
- Кожна задача — компонент `<TaskRow />` (див. п.6.1).
- FAB-мікрофон знизу.

### 5.2. Upcoming (`/upcoming`)

- TopBar: заголовок «Upcoming».
- Вертикальний скрол з групами по днях:
  - Заголовок групи: великий номер числа + назва дня тижня + (якщо сьогодні) сірий бейдж `Today`.
  - Усередині групи — задачі (з тим же `<TaskRow />`).
- Дні без задач — не показуємо.
- Без табів, без бокових перемикачів.

### 5.3. Inbox (`/inbox`)

- Список задач, які AI не зміг впевнено розпарсити (наприклад, без дати) — або ті, що додав користувач вручну без дати.
- Той самий `<TaskRow />`.

### 5.4. Anytime / Someday / Logbook (`/anytime`, `/someday`, `/logbook`)

- У MVP — стрічка задач без груп.
- Logbook — задачі зі статусом `done`, сортування за датою виконання.

## 6. Ключові компоненти

### 6.1. `<TaskRow />`

- Висота не фіксована, padding `py-3`.
- Структура зліва направо:
  - **Чекбокс** (24×24, `rounded-md`, бордер `text.tertiary`). При відмітці — заливка `accent.green`, біла галочка, текст задачі блякне (`opacity-50 line-through`).
  - Контент:
    - Назва задачі (`text-[17px]`).
    - Під назвою — проєкт/контекст сірим (`text-[13px] text-secondary`), якщо є.
  - Праворуч — індикатори: іконка нотатки (`FileText`), іконка чек-листа (`ListChecks`), бейдж дедлайну `🚩 3d left` (червоний для прострочених).
- Жести:
  - **Тап** → інлайн-розкриття картки задачі (`<TaskCardExpanded />`).
  - **Свайп вправо** → відмітити виконаним (анімація заливки зеленим, потім зникає в Logbook).
  - **Свайп вліво** → відкриває кнопки `Schedule`, `Delete` (червона).
- Реалізація свайпів — `framer-motion` `drag="x"` з порогами.

### 6.2. `<TaskCardExpanded />`

Інлайн-розгортання прямо в списку (не модалка):

- Зʼявляється під рядком, висота анімовано росте.
- Поля:
  - Назва (редагована inline).
  - Notes (`textarea`, плейсхолдер «Нотатки»).
  - Sub-checklist (опційно): рядки з міні-чекбоксами, кнопка «+ Додати підпункт».
  - Рядок дій: `📅 When`, `🏷 Tag`, `☰ Checklist`, `🚩 Deadline`.
- Тап поза карткою → згортається з автозбереженням.

### 6.3. `<FabMic />`

- `fixed bottom-6 right-5`, 64×64, `rounded-full`, фон `accent.blue`, іконка `Mic` біла 28px.
- При тапі — відкриває `<VoiceModal />`.
- Hover/active — легкий scale-down (0.95).

### 6.4. `<VoiceModal />` — голосовий brain dump

Повноекранна модалка (`Dialog` на весь viewport).

- Фон — темний або gradient blue→black, контраст з білим UI решти аппки.
- Центр екрану — **велике коло з пульсуючою хвилею** (анімація амплітуди реагує на гучність мікрофона; для MVP — синтетична хвиля, але каркас компоненту має приймати `amplitude` пропом).
- Над колом — таймер запису (`00:12`).
- Під колом — текст-плейсхолдер: «Слухаю… розкажи все, що треба сьогодні зробити».
- Унизу — дві великі кнопки:
  - **Стоп** (білий круг із квадратом усередині) — завершує запис, переходить до екрану AI-обробки.
  - **Скасувати** (текст «Скасувати» сірим).
- Свайп вниз або тап по «Скасувати» — закриває модалку без збереження.

### 6.5. `<AiProcessing />` екран-перехід

- Між зупинкою запису і превʼю задач.
- Просте центроване повідомлення: «AI розбирає твій інпут…» + анімований spinner / пульс.
- Зникає, щойно `/api/parse` повертає результат.

### 6.6. `<TasksPreview />` — превʼю згенерованих задач

- Список карток із результатами AI:
  - Назва задачі (редагована inline).
  - Бейджі: **пріоритет** (P1 червоний / P2 помаранчевий / P3 сірий), **оцінка часу** (`~30 хв`), **дедлайн** (`Сьогодні`, `Завтра`, `Пт`).
  - Тап по картці → можна виправити поля.
  - Свайп вліво → видалити цю задачу з превʼю.
- Унизу — дві кнопки на повну ширину:
  - **Додати все в Today** (основна, синя).
  - **Розкидати по датах** (вторинна, бордер) — використовує дедлайни, які витягнув AI.

## 7. AI-флоу (контракт API)

### 7.1. `/api/transcribe` (POST)

- Вхід: `multipart/form-data` з аудіо (webm/opus з браузера через `MediaRecorder`).
- Бек: викликає Whisper-сумісний ендпоінт (наприклад через Anthropic-сумісну проксі або OpenAI Whisper API — ENV ключем).
- Вихід: `{ text: string }`.

### 7.2. `/api/parse` (POST)

- Вхід: `{ text: string, now: ISOString }`.
- Бек: викликає Claude (модель `claude-sonnet-4-6` або новішу — конфіг через ENV) з системним промптом, що повертає **строго JSON-масив** задач:
  ```ts
  type ParsedTask = {
    title: string;
    priority: "P1" | "P2" | "P3";
    estimateMinutes: number | null;
    deadline: string | null; // ISO date
    notes?: string;
  };
  ```
- Промпт має враховувати поточний час/дату, мову інпуту (UA/EN), пояснювати правила пріоритезації.
- Вихід: `{ tasks: ParsedTask[] }`. Якщо парсинг не вдався — `{ tasks: [], raw: string }` (інпут залишається в Inbox як одна задача).

### 7.3. ENV

Файл `.env.local` (приклад):

```
ANTHROPIC_API_KEY=...
OPENAI_API_KEY=...        # для Whisper, опційно
```

На Vercel — ті самі змінні в Project Settings → Environment Variables.

## 8. Стан і збереження даних

- **Без бекенду** в MVP: усе в `zustand` + персист у `localStorage`.
- Структура стану:
  ```ts
  type Task = {
    id: string;
    title: string;
    notes?: string;
    priority: "P1" | "P2" | "P3";
    estimateMinutes: number | null;
    deadline: string | null;
    when: "today" | "upcoming" | "anytime" | "someday" | "inbox";
    status: "open" | "done";
    createdAt: string;
    completedAt?: string;
    subtasks?: { id: string; title: string; done: boolean }[];
  };
  ```
- Стор-методи: `addTasks`, `toggleDone`, `update`, `remove`, `moveTo(when)`, `setDeadline`.

## 9. Жести і анімації — підсумок

- Drawer: slide-in 250ms easeOut, overlay fade-in.
- Task expand: висота + opacity 200ms.
- Свайп задачі: drag по X з порогами ±80px, snap-back з spring.
- Чекбокс: scale-up 1.0→1.15→1.0 + плавна заливка.
- VoiceModal: fade + scale-up з центру, пульс кола 1.5s loop.

---

## 10. Mobile-first вимоги

- Стартова viewport: 390×844 (iPhone 14). Усі екрани мають коректно виглядати від 360px ширини.
- Жодних `hover`-залежних інтеракцій без тач-альтернативи.
- Десктоп: контейнер центрується (`max-w-md mx-auto`), решта екрану — сіра «рамка», як у мобільному превʼю. Це не повноцінний десктоп.
- `<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, viewport-fit=cover">`.

## 11. MVP — чекліст приймання

Все нижче має працювати наживо на Vercel-домені:

1. Головний екран `/` (Today) з мокованими задачами при першому запуску.
2. Гамбургер відкриває drawer на 50% ширини; усі 6 пунктів навігують на відповідні екрани.
3. Тап на задачу — інлайн-розкриття; свайп вправо — done; свайп вліво — delete.
4. FAB-мікрофон відкриває VoiceModal на весь екран з пульсуючим колом і таймером.
5. Запис аудіо через `MediaRecorder` → відправка на `/api/transcribe` → текст.
6. Текст → `/api/parse` → масив задач → `<TasksPreview />`.
7. «Додати все в Today» додає задачі в стор і повертає на Today.
8. Стан зберігається в `localStorage` між перезавантаженнями.
9. Деплой на Vercel, ENV налаштовані, посилання працює.

## 13. Перші кроки для Claude Code

1. `npx create-next-app@latest` з App Router + TS + Tailwind.
2. Поставити shadcn/ui, framer-motion, lucide-react, zustand.
3. Конфіг Tailwind: додати кастомні кольори з п.2.1, шрифти з п.2.2.
4. Підняти layout shell (TopBar + FAB + Drawer + Modal-портал).
5. Замокати 5–7 задач для Today / Upcoming, щоб одразу побачити UI.
6. Зробити `<TaskRow />` повністю (включно зі свайпами).
7. Drawer + навігація між екранами.
8. VoiceModal + `MediaRecorder` (поки без AI — лог blob у консоль).

Працюємо ітеративно: спочатку видимий каркас з мок-даними, потім AI, потім поліровка анімацій.
